\paragraph{Cross-benchmark deltas (Level 2).} The cross-benchmark level keeps the
same backbone but replaces the harness with a pure-vision loop using the
backbone's native \texttt{mobile\_use} tool-call format (no accessibility
channel). It evaluates on the 110-task MobileWorld $T_{\text{eval}}$ and adapts on
the 51-task $T_{\text{adapt}}$ (Table~\ref{tab:mw-coverage}); there is no seed
dimension ($K{=}1$). The static prior carried into the adaptation rungs is the
app-granular LAYER~2 plus category $L_C$ configuration with entry-level retrieval
(only the $\le 3$ task-relevant archetypes per block are injected). The weight
channel uses the same group-relative update with a small KL anchor to the frozen
reference ($\beta{=}10^{-3}$).

\section{LAYER-2 mutation operator}
\label{app:mutation}

The mutation step in Algorithm~\ref{alg:evofsmrl} is implemented as a two-step
prompt to the frozen reference $\pi_{\text{ref}}$. Step~1 builds a structured
reflection: the FSM in JSON form, the recent rollout trajectories under the best
variant (action sequence, screen tags, reward), and a directive to identify
(a)~which abstract step or verification check appears to have been violated in
failing rollouts and (b)~which step appears to have been load-bearing in
successful rollouts. Step~2 takes that reflection and the FSM and asks for a
JSON-typed diff over LAYER~2 only (operations: \texttt{add\_failure\_mode},
\texttt{add\_verification\_check}, \texttt{rewrite\_abstract\_step},
\texttt{add\_category}, and a small set of removals). The diff is validated
against the FSM schema and lint-checked for app-specific tokens before being
applied; rejected diffs are logged and do not enter the population.
