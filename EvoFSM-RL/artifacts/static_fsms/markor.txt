CATEGORY: ADD_ENTRY
  precondition: App is open at a list/index view that exposes an affordance to create a new item
  abstract_steps:
    1. Invoke the primary create-new affordance (commonly a prominent action button)
    2. Provide the required identifier in the creation dialog, clearing any prefilled placeholder first
    3. Select the correct item kind/format if the dialog offers a choice
    4. Confirm creation
    5. If opened in an editor, enter the required body content
    6. Persist the change (explicit commit if auto-persistence is not guaranteed)
  failure_modes:
    - 'Leaving the prefilled placeholder identifier concatenated with the intended identifier'
    - 'Not switching item kind (leaf vs container) before confirming'
    - 'Garbled extension/suffix field producing wrong format'
    - 'Declaring success before persisting'
    - 'Creating item in wrong parent container'
  verification_checklist:
    - 'New item appears in the listing with the exact expected identifier'
    - 'Opening the item shows the expected body content'
    - 'No stray/extra items created'
    - 'Correct parent container'

CATEGORY: REMOVE_ENTRY
  precondition: Target item is visible in a list view
  abstract_steps:
    1. Long-press the target row to enter selection mode
    2. Invoke the removal action from the contextual action bar
    3. Confirm removal in the confirmation dialog
  failure_modes:
    - 'Selecting wrong row (similar identifiers)'
    - 'Missing the confirmation step'
    - 'Removing parent container instead of the target item'
  verification_checklist:
    - 'Item no longer appears in the listing'
    - 'No confirmation dialog left open'
    - 'Sibling items intact'

CATEGORY: BULK_REMOVE
  precondition: Multiple items to clear from a list
  abstract_steps:
    1. Repeat the single-item removal flow for each item, OR use multi-select if available
    2. After each removal, verify return to list before selecting the next target
  failure_modes:
    - 'Index shifting after removal leading to wrong next item'
    - 'Stopping before all targets are cleared'
  verification_checklist:
    - 'Listing contains only non-target items (often empty)'

CATEGORY: EDIT_ENTRY_CONTENT
  precondition: Target item exists and can be opened in an editor
  abstract_steps:
    1. Open the item from the list
    2. Position cursor at the required location (start, end, or full-content selection)
    3. Insert, append, prepend, or replace text as required
    4. Persist the change
  failure_modes:
    - 'Appending instead of prepending (or vice versa)'
    - 'Not selecting the full existing content before replacing (causing concatenation)'
    - 'Forgetting trailing newline when prepending'
    - 'Not persisting before navigating away'
  verification_checklist:
    - 'Item body matches expected content exactly'
    - 'Line breaks and ordering match specification'
    - 'Changes persisted (indicator dismisses/returns to list cleanly)'

CATEGORY: RELABEL_ENTRY
  precondition: Item exists in a list view that supports relabeling
  abstract_steps:
    1. Long-press the item to enter selection mode
    2. Choose the relabel action from the contextual action bar
    3. Enter the new identifier in the dialog and confirm
  failure_modes:
    - 'Attempting to relabel from within the editor (title click does not relabel)'
    - 'Typing into the editor title field and assuming it relabeled the item'
    - 'Leaving old extension/suffix when a new one is required'
  verification_checklist:
    - 'List shows the item with the new identifier'
    - 'Old identifier no longer present'
    - 'Content preserved'

CATEGORY: RELOCATE_ENTRY
  precondition: Source item and destination container both exist
  abstract_steps:
    1. Long-press the source item to select it
    2. Open the contextual overflow and choose the relocate action
    3. In the destination picker, navigate up and/or into the target container
    4. Confirm with the 'choose this container' action
  failure_modes:
    - "Selecting the source's own container as destination"
    - 'Stopping at the picker without a final confirmation'
    - 'Not navigating up to the common ancestor first'
  verification_checklist:
    - 'Item no longer present in source container'
    - 'Item present in destination container'
    - 'Content unchanged'

CATEGORY: NAVIGATE_TO
  precondition: Target app/screen may not be on current screen
  abstract_steps:
    1. If target app not visible, open the system launcher or app list to switch context
    2. Traverse the container hierarchy via click-in and back-navigation
    3. Verify breadcrumb/title matches target
  failure_modes:
    - 'Confusing similar container identifiers'
    - 'Getting stuck in side menus (e.g., configuration panes) that do not lead to target'
    - 'Repeating the same failing click'
  verification_checklist:
    - 'Breadcrumb/title matches expected path'
    - 'Expected content visible'

CATEGORY: CROSS_APP_TRANSFER
  precondition: Source content is prepared; destination app is installed
  abstract_steps:
    1. Create/finalize source content in the origin app and persist it
    2. Launch the destination app via the launcher
    3. Initiate a new message/record in the destination app
    4. Provide the recipient/target identifier
    5. Insert the content (enter it directly or transfer via system transfer buffer)
    6. Send/commit
  failure_modes:
    - 'Opening the wrong destination app (a similar but different communication app)'
    - 'Not persisting source before switching apps'
    - 'Leaving the draft unsent'
    - 'Using a pre-existing contact instead of the required recipient identifier'
  verification_checklist:
    - 'Destination app shows the sent/committed item with exact content'
    - 'Recipient identifier matches'
    - 'No residual draft'

CATEGORY: MERGE_ENTRIES
  precondition: Multiple source items and a specified destination identifier
  abstract_steps:
    1. Create the destination item first (empty)
    2. For each source in order: open, select full content, capture it, return to destination, insert it, append a separator if required
    3. Persist destination
  failure_modes:
    - 'Getting stuck trying to create destination from editor menus instead of the list view'
    - 'Losing captured content between app switches'
    - 'Wrong order'
    - 'Missing separators between sections'
  verification_checklist:
    - "Destination contains each source's content in the required order"
    - 'Separators (newlines) between sections'
    - 'Source items unchanged'

CATEGORY: TRANSCRIBE_FROM_MEDIA
  precondition: Media source and transcription target available
  abstract_steps:
    1. Open the media player and start playback
    2. Observe each piece of text/frame
    3. Switch to the transcription editor and record the observation with the required formatting
    4. Repeat until media ends
    5. Persist the transcription
  failure_modes:
    - "Using 'wait' without switching back to the media app"
    - 'Losing place when switching apps repeatedly'
    - 'Fabricating content not actually shown'
    - 'Incorrect delimiter formatting'
  verification_checklist:
    - 'Transcription contains the complete ordered sequence'
    - 'Formatting (delimiter, identifier) matches spec'
    - 'Transcription persisted'

CATEGORY: QUERY_DATA
  precondition: Source of the requested data is reachable in the app
  abstract_steps:
    1. Navigate to the screen showing the requested data
    2. Read/extract the required field(s)
    3. Report back without altering data
  failure_modes:
    - 'Modifying data while reading'
    - 'Reporting stale/cached values'
  verification_checklist:
    - 'Reported value matches on-screen value'
    - 'No side effects on data'
