CATEGORY: CREATE_ENTRY
  precondition: App launched and main create/compose surface reachable
  abstract_steps:
    1. Navigate from launcher to the app's primary surface
    2. Pass through onboarding/welcome if present
    3. Confirm or accept default configuration if an initial configuration screen appears
    4. Dismiss any transient informational dialogs
    5. Trigger the primary create action (e.g. record/start)
    6. Allow the creation process to run for the required duration
    7. Trigger stop/finalize action
    8. Confirm in any resulting finalize dialog
  failure_modes:
    - 'Clicking indices not present in the current UI listing'
    - 'Mistaking a selector list for an editable text field'
    - 'Ignoring or missing a blocking dialog'
    - 'Terminating before the finalize step'
  verification_checklist:
    - 'A new entry with non-zero size/duration appears in the list or main view'
    - 'Timer/status returns to idle state'
    - 'No blocking dialog remains on screen'

CATEGORY: CREATE_ENTRY_WITH_CUSTOM_NAME
  precondition: App provides a means to assign a custom label, either at finalization time or via pre-set options
  abstract_steps:
    1. Perform CREATE_ENTRY up to the finalize step
    2. When a finalize/name dialog appears, clear any pre-filled text
    3. Type the desired custom name
    4. Confirm finalization
  failure_modes:
    - 'Assuming a pre-creation name selector allows arbitrary text (it may only offer templates)'
    - 'Trying to rename from a list view that lacks rename affordance'
    - 'Getting stuck repeating the same invalid click instead of re-examining UI'
    - 'Missing the finalize dialog because stop leads directly to an intermediate state'
  verification_checklist:
    - 'Final entry appears with exactly the requested name'
    - 'No duplicate entry with default name is left behind'
    - 'Entry has expected size/duration'
