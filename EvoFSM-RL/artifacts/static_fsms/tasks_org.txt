CATEGORY: QUERY_INFO
  precondition: Target data-bearing screen reachable from launcher; query specifies a filter (date, status, priority, etc.)
  abstract_steps:
    1. Launch the target app from the launcher
    2. Navigate to the primary list/overview screen
    3. Resolve any relative terms in the query (today, next week, after next) to absolute values using the device/task context
    4. Scroll through the full list to enumerate all candidate items
    5. Apply the query's filter predicate to each item using observable visual attributes
    6. Aggregate matching items into the requested output format (list, count, yes/no)
    7. Emit a single answer action, then immediately emit a terminal completion signal
  failure_modes:
    - 'Confusing relative date labels with the queried absolute date'
    - 'Answering from a partial view without scrolling the full list'
    - 'Repeating the same answer action without terminating, causing step exhaustion'
    - 'Guessing semantic attributes (priority, category) from unrelated visual cues'
    - 'Declaring the query infeasible before exhausting available navigation (scroll, open detail)'
    - 'Miscounting due to duplicated or partially visible rows after scrolling'
  verification_checklist:
    - 'Did I reach the list screen that actually contains the data?'
    - 'Did I map all relative terms to concrete values consistent with the current date?'
    - 'Did I scroll until the end of the list (no new content appears)?'
    - 'Does each returned item satisfy every part of the filter (date AND status AND priority)?'
    - 'Is the answer format exactly what the user requested (comma-separated / count / None)?'
    - 'Did I emit a terminal completion action exactly once after answering?'

CATEGORY: FILTER_BY_ATTRIBUTE
  precondition: List view exposes per-item visual attributes (color, icon, label) that encode the attribute of interest
  abstract_steps:
    1. Identify the concrete visual encoding of the target attribute on this app (e.g., checkbox color = priority)
    2. Iterate items and apply the encoding rule
    3. Return matching titles/identifiers
  failure_modes:
    - "Assuming a default encoding from other apps rather than observing this app's cues"
    - 'Treating urgency/date as priority when priority is encoded separately'
  verification_checklist:
    - 'Is the attribute encoding confirmed by at least one unambiguous example?'
    - 'Are negative examples (items that should NOT match) excluded?'

CATEGORY: NAVIGATE_TO
  precondition: Target app installed; currently elsewhere
  abstract_steps:
    1. Invoke the app-launch primitive with the app name
    2. Verify the landing screen matches expectations before acting
  failure_modes:
    - 'Acting before the landing screen is fully loaded'
  verification_checklist:
    - 'Title bar/landmark of destination screen is visible'
