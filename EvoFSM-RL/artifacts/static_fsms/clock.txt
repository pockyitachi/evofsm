CATEGORY: ENTER_STRUCTURED_VALUE
  precondition: An input surface exists that accepts a multi-field structured value (e.g., H:M:S, date parts) via a keypad that shifts digits
  abstract_steps:
    1. Navigate to the input surface
    2. Understand the keypad semantics: determine whether digits fill from the right (shift-left) or whether fields are edited independently
    3. If shift-fill: enter the full digit sequence in order from most-significant to least-significant, including leading zeros to place digits in correct fields
    4. Avoid partial backspaces unless correcting the last digit only
    5. Verify the displayed value matches the target exactly before confirming
  failure_modes:
    - 'Assuming each field is edited independently when keypad actually shifts digits across fields'
    - 'Entering only significant digits and skipping leading zeros, causing digits to land in the wrong field'
    - "Using backspace repeatedly to 'clear' without realizing it only removes one digit at a time"
    - 'Re-entering after partial clear and compounding misalignment'
    - 'Declaring completion before verifying display matches target'
  verification_checklist:
    - 'Displayed value exactly equals target across all sub-fields'
    - 'No leading/trailing unintended digits'
    - 'Did not inadvertently trigger an activation/confirm control'
