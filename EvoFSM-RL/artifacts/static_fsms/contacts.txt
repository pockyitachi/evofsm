CATEGORY: ADD_ENTRY
  precondition: Application opened to a list/overview screen with an affordance to create a new item
  abstract_steps:
    1. Navigate from launcher to the target application's list screen
    2. Invoke the create-new-item affordance (floating action button or explicit 'Create' entry)
    3. Focus each required input field in order and enter the corresponding value
    4. Confirm by activating the primary commit control
    5. Verify the new item appears in the list
  failure_modes:
    - 'Committing before all required fields are filled'
    - 'Typing into an unfocused field'
    - 'Missing the create affordance because it is off-screen or behind a tab'
  verification_checklist:
    - "New item's identifier is visible in the list view"
    - 'No error/validation banner present'
    - 'Return to list state (not still in editor)'

CATEGORY: EDIT_ENTRY_FIELD_WITH_ENUM_LABEL
  precondition: Editor form open with a field that has an associated enum label selector
  abstract_steps:
    1. Fill primary value field with input_text
    2. Activate the adjacent label dropdown/selector
    3. Choose the desired label from the option list
    4. Confirm the label field reflects the chosen value
  failure_modes:
    - 'Clicking the value field instead of the dropdown trigger'
    - 'Selecting wrong option because list indices shift after opening'
  verification_checklist:
    - 'Label field text matches requested value'
    - 'Value field retained its entered text'

CATEGORY: CREATE_DRAFT_WITHOUT_COMMIT
  precondition: Editor form reachable and task specifies not to persist
  abstract_steps:
    1. Populate all specified fields
    2. Adjust any optional attributes (labels/types) as specified
    3. Stop before invoking the primary commit control
    4. Declare task complete while editor still shows draft state
  failure_modes:
    - 'Accidentally activating the primary commit control'
    - 'Navigating away and losing the draft'
  verification_checklist:
    - 'All requested values are visible in their fields'
    - 'Editor screen is still active (not returned to list)'
    - 'No success confirmation toast appeared'

CATEGORY: NAVIGATE_TO
  precondition: Launcher or unrelated screen visible
  abstract_steps:
    1. Identify the target app icon or use open_app by name
    2. If app opens to a non-target subsection, switch tabs/drawer to reach target section
  failure_modes:
    - 'App icon not on current launcher page'
    - 'Landing on wrong tab inside a multi-tab app'
  verification_checklist:
    - 'Target section header/title visible'
    - 'Expected primary affordances (e.g., create button) present'
