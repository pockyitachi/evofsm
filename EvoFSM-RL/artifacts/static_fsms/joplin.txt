CATEGORY: QUERY_ITEM_ATTRIBUTE
  precondition: Target item is identified by name; app may not be in foreground
  abstract_steps:
    1. Launch the relevant app from the launcher
    2. Navigate to the named item by browsing the list or using an in-app lookup affordance
    3. Inspect the visible marker/attribute (icon, prefix, checkbox) that indicates the attribute in question
    4. Return a boolean/categorical answer based on the marker
  failure_modes:
    - 'Declaring the task infeasible from the launcher without attempting to open the app'
    - 'Answering based on a screen that does not actually contain the target item'
    - 'Looping on the same answer action without finalizing with a completion status'
  verification_checklist:
    - "The target item's row is actually visible on screen"
    - 'The attribute marker is unambiguously present or absent'
    - 'A single completion/status signal is emitted after the answer'

CATEGORY: COUNT_ITEMS_BY_ATTRIBUTE
  precondition: A container (folder/list/category) and a filtering attribute are specified
  abstract_steps:
    1. Open the app and navigate to the specified container via its navigation drawer/menu
    2. Enumerate all list entries, scrolling as needed to cover the full list
    3. Count entries whose row carries the filtering attribute marker
    4. Return the count as a number
  failure_modes:
    - 'Counting within the wrong container because the app opened to a different default'
    - 'Missing entries that require scrolling'
    - 'Confusing unrelated visual markers with the target attribute'
  verification_checklist:
    - 'Current container label matches the requested one'
    - 'List has been scrolled to end or confirmed fully visible'
    - 'Only entries with the specific attribute marker are counted'

CATEGORY: LOOKUP_FIELD_IN_ENTRY
  precondition: Entry title and desired field/sub-item are specified
  abstract_steps:
    1. Open the app and invoke its entry-lookup affordance
    2. Provide the entry title and select the matching result
    3. Locate the section containing the field and read its value
    4. Return the value verbatim
  failure_modes:
    - 'Skipping app launch and declaring infeasible'
    - 'Opening a wrong result with a similar title'
    - 'Reading from the wrong section of the entry'
  verification_checklist:
    - 'Opened entry title matches the requested one'
    - 'Value is read from the correct labeled section'
    - 'Answer preserves units/qualifiers as shown'

CATEGORY: NAVIGATE_TO_CONTAINER
  precondition: Multiple containers exist; a specific one is required
  abstract_steps:
    1. Open the navigation drawer/menu
    2. Locate the named container (scroll if necessary)
    3. Tap it to switch context
  failure_modes:
    - 'Assuming the currently opened container is the target'
    - 'Missing the target entry because it resides in a different container'
  verification_checklist:
    - 'Header/title of the list reflects the requested container name'
